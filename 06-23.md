# 验收题目

## CSS 3D 加速是什么？什么情况下会启动 3D 加速？可能带来什么问题？

- 定义：3d 加速即 gpu 加速，指的是某些元素利用 gpu 加速的条件，以此单独形成一层自己的合成层，重绘只会重绘本身
- 条件：translate/opcaity/will-change/filter
- 问题：
  - 兼容性问题
  - 功耗问题：会增加计算机和设备的功耗
  - 内存问题：会调动更多的系统内存和显存资源，可能造成层爆炸的情况（解决办法：加一个 z-index 属性，避免层重叠）

## 浏览器为什么默认需要为每一个 tab 开启一个进程？可能带来什么好处与劣势？

- 原因：是为了实现多进程架构
- 好处：
  - 隔离性强：一个标签页的崩溃或内存泄漏不会影响其他标签页的稳定性，还助于防止恶意网站通过攻击浏览器进行用户隐私泄露或系统安全性受损。
  - 并行处理：多进程架构允许浏览器同时处理多个任务
  - 资源管理：个进程可以独立分配和管理系统资源，避免一个标签页过度占用资源导致整个浏览器变慢或崩溃
- 坏处：
  - 内存占用增加：每个进程都需要一定的内存来运行，大量的标签页，浏览器可能消耗大量的内存资源
  - 进程间通信开销：不同进程之间需要进行通信和同步，这会引入一定的开销，尤其是在需要频繁进行数据交换或共享状态的情况下
  - 启动时间增加：每个标签页都需要一个独立的进程来运行，这会导致浏览器的启动时间增加。

## 浏览器中，为什么说 JS 线程与渲染线程是互斥的？

- 原因：因为 js 和渲染线程同属于主线程， js 会阻碍渲染流程，只有等到 js 执行完成之后，才会继续渲染

## 如何优化页面渲染性能？

- 页面渲染优化：
  - 优化 html 代码：
    - js 链接放底部
    - css 链接放顶部
    - 减少 DOM 层级
  - 优化 js、css 代码
    - 减少重拍重绘操作
      - 涉及到虚拟 DOM：虚拟 DOM 的好处就是不在浏览器页面进行增删改，从而避免了很多的重绘回流操作
      - display：none/fregament
      - 绝对定位，脱离文档流
      - 懒加载
      - script 标签的 derfer 和 async
    - 降低 css 选择器复杂性
  - 动画效果优化
    - 使用 requestAnimationFrame
      - 思考点：
        - 如何计算动画帧率？（看 1 秒钟内执行了多少次回调函数）
    - 通过 animation\translate\will-change 属性\来实现硬件加速
